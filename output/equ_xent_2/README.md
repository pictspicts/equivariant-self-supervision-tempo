# 学習録: Hybrid Loss + 無劣化タイムストレッチ (Super-Sampling & Random Crop)

## 現在の設定
* **実験名**: `equ_xent_2`
* **Loss設定**: `hybrid` (等変性損失 ＋ シフト交差エントロピー)
* **データ拡張手法**: 無劣化画像ストレッチ ＋ 並進不変性（Translation-Invariance）獲得クロップ
* **バッチサイズ**: 64

## 実施されている推測対策の完全版

### 1. 無音パディング／クロップによるヒント漏洩の完全遮断（NEW!）
* **問題点**: タイムストレッチの処理において、「曲が速くなった時に余った無音パディングの量」や「曲が遅くなった時に切断された箇所の不自然さ」そのものが、AIにストレッチ倍率 $r$ を教える特大のカンニングペーパーになっていました（これによって前回全曲が特定定数へモード崩壊しました）。
* **解決策**: データローダーの読み込み長さを **1.3倍（780000サンプル: 約17.6秒分）に増量（スーパーサンプリング）** しました。この巨大なメルスペクトログラム画像をストレッチし、その中から **「完全に音楽で埋まっている元の要求サイズ（1361フレーム: 約13.6秒分）の窓」をランダムな位置でスライドさせてくり抜く（Random Crop）処理** に変更しています。これにより無音の余白や切断面が物理的に一切存在しなくなり、AIはストレッチ量を推測不可能になりました。

### 2. 並進不変性 (Translation-Invariance) の獲得
* 副産物として、同じ元の曲から生成される2つのスペクトログラム ($x_i$ と $x_j$) は「切り抜かれる小窓の位置（曲のフレーズ）」が独立してズレるようになりました。これにより、「ボーカルやフレーズの位置が数秒違っていても、元の曲のテンポ設定は同じである」という堅牢な知見（並進不変性）をAIが自動的に学習するため、BYOL等の手法と同様の非常に理想的なデータ拡張となっています。

### 3. 画像ベース・タイムストレッチへの移行
* 不自然な劣化ノイズ（アーティファクト）を残すフェーズボコーダ (`torchaudio` 波形ストレッチ) を廃止し、対数メルスペクトログラム画像に対して `F.interpolate` を使った「時間軸方向への直接画像リサイズ」に入れ替えました。音質のノイズ・劣化ヒントも完全に遮断されています。

### 4. ハイブリッド・ロス (Log-Equiv + Shift-Xent)
* 確率をゼロに寄せようとする「スケール崩壊」を防ぐ対数等変性ロスと、分布を尖らせる正則化を担うシフト交差エントロピーを統合。さらに `.detach()` による動的バランシングによって、両者が常に 5:5 の完全な影響力で綱引きするように設計しています。
