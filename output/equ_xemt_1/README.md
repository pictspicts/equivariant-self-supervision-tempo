# 学習録: ハイブリッドロス (Equivariance + Shift-Xent) テスト

## 現在の設定
* **Loss設定**: `hybrid` (等変性損失 ＋ シフト交差エントロピー)
* **バッチサイズ**: 64

## 実行中の損失関数の構成と理論

### 1. 対数-等変性損失 (Log-Equivariance Loss)
* **目的**: タイムストレッチの比率に応じて、出力されるテンポ確率分布が正しく平行移動（シフト）するかを強制する。
* **モード崩壊対策**: 出力分布をZ変換的にスカラー値 ($\phi$) に圧縮するだけでなく、両辺の対数（$\log$）を取ることで絶対的なスケールを無視。モデルが確率を端に寄せてゼロを作ろうとするズル（モード崩壊）を数学的に無効化し、純粋なシフト量 ($k \cdot \log(\alpha)$) のみを学習させます。

### 2. シフト交差エントロピー (Shift-Xent / 正則化項)
* **目的**: 上記の等変性ロスだけでは「分布の山のカタチ」が崩れても（山が2〜3つあっても）パスしてしまうため、分布全体が一つの鋭いピークとして尖って移動するように、クロスエントロピーを用いて形状の正則化（Shape Regularization）を行います。

### 3. 5:5 動的バランシング (Dynamic Weighting)
* **目的**: 計算スケールが根本的に異なる2つのロスが、互いに同等の圧力でネットワークを学習させるための仕組み。
* **手法**: 各ロス値の `.detach()`（勾配を切ったただの数値）を用いて、`Equiv / Xent` の比率係数 $w$ をリアルタイムに算出。これをクロスエントロピー側にのみ掛け合わせることで、各バッチごとに必ず「等変性ロスのスケール」と「正則化ロスのスケール」が 1:1 (5:5) になるよう自動調整しています。
