# PESTOにおける3つの損失関数と全体の学習の仕組み

PESTOモデルでは、ラベルなしの自己教師あり学習を成功させるために、以下3つの損失関数を組み合わせて学習を行っています。

## 1. 等変性損失 (Equivariance loss: $L_{equiv}$)
* **目的**: 入力音声のピッチをシフトさせた分だけ、出力される確率分布も正確に同じ量だけシフト（等変）しているかを評価します。
* **計算方法**: 確率分布の各要素に固定値 $\alpha$ のべき乗を掛けて足し合わせる特殊な決定論的線形形式 $\phi(y)$ を用います。元の分布の計算結果 $\phi(y)$ と、$k$ 半音シフトさせた分布の計算結果 $\phi(y^{(k)})$ を割り算し、その比率が理論上の比率である $\alpha^k$ と一致するかどうかをHuber（フーバー）損失で計算します。
* **役割**: モデルが「すべての入力に対して同じ平坦な分布を返す」といった学習のサボり（崩壊：Collapse）を起こすのを防ぐ、モデルの中核的な役割を果たします。

## 2. 正則化損失 / シフト交差エントロピー (Regularization loss: $L_{SCE}$)
* **目的**: 等変性損失の「計算上の抜け道（分布が崩れていても数値の比率だけ合ってしまう状態）」を塞ぎ、確率分布の「山の形」が崩れずに正しく平行移動するよう強く強制（正則化）します。
* **計算方法**: 元の分布のピッチ $i$ の確率と、シフト後の分布におけるピッチ $i+k$ の確率を1対1で突き合わせ、交差エントロピー誤差（Shifted Cross-Entropy）を全音域で計算し合計します。
* **役割**: 分布全体が連動して動いているかをピンポイントで厳密に答え合わせします。この損失を除去すると精度が致命的に低下するため、等変性損失とセットで不可欠な要素です。

## 3. 不変性損失 (Invariance loss: $L_{inv}$)
* **目的**: 音色やノイズなど、ピッチ以外の要素の変化に対してモデルの予測結果がブレないようにします。
* **計算方法**: 元の音声から予測した確率分布 $y$ と、ピッチを変えずにホワイトノイズ追加や音量（ゲイン）変更などのデータ拡張を施した音声から予測した確率分布 $\tilde{y}$ の間で、通常の交差エントロピーを計算します。
* **役割**: ノイズが加わっても元の音声と「まったく同じ確率分布」を出力するように強制することで、純粋にピッチ情報だけを捉えさせます。また、未知のデータに対するオクターブエラーを防ぎ、モデルの汎化性能を向上させる効果があります。

## 4. 全体の損失関数 (Total Loss)
* 上記3つの損失関数は、最終的に足し合わせて最適化されます（$L = \lambda_{inv} L_{inv} + \lambda_{equiv} L_{equiv} + \lambda_{SCE} L_{SCE}$）。
* **重みの自動調整**: 各損失の割合を決定する係数（$\lambda$）は人間が手動で固定するのではなく、勾配ベースの手法（gradient-based method）を用いて、学習の進行に合わせてシステムが動的かつ最適なバランスになるように自動調整を行います。
